## *Padding*

O modo [CBC](https://cryptography.io/en/latest/hazmat/primitives/symmetric-encryption/#cryptography.hazmat.primitives.ciphers.modes.CBC) pressupõe a utilização de um esquema de *padding*, como o [PKCS7](https://cryptography.io/en/latest/hazmat/primitives/padding/#cryptography.hazmat.primitives.padding.PKCS7). 

Para se visualizar o efeito de um mecanismo como o PKCS7, atente-se ao seguinte fragmento de código:

```python
BLOCK_SIZE = 128

def pad(ptxt):
    padder = padding.PKCS7(BLOCK_SIZE).padder()
    padded = padder.update(ptxt) + padder.finalize()
    return padded

def ints(bstr):
    return [int(x) for x in bstr]

ints(pad(b"abc"))
```

Um aspecto importante dos mecanismos de *padding* é que estes introduzem redundância. Desde logo, porque o resultado é sempre múltiplo do tamanho do bloco. Mas as próprias regras do mecanismo determinam uma certa forma do resultado. Nas prática, quando "retiramos" o *padding* a uma mensagem, pode acontecer que o resultado seja a sinalização que existe uma violação do formato previsto, como no exemplo que se segue:

```python
def unpad(padded):
    unpadder = padding.PKCS7(BLOCK_SIZE).unpadder()
    ptxt = unpadder.update(padded) + unpadder.finalize()
    return ptxt

# removendo um bytes...
unpad(pad(b"abc")[:15])
```

## *Padding oracle*

Quando o *padding* é combinado com uma cifra (e.g. AES/CBC), este é aplicado antes da operação de "cifrar", e removido depois da operação "decifrar". A questão é que, se existir um erro na operação de *unpadding*, a operação de "decifrar" falha, e esse facto é normalmente **observável** por um adversário. Podemos emular esse conhecimento do adversário por o que se designa por um **_padding oracle_**:

```python
def pad_orcl(ctxt):
    padded_msg = dec(ctxt) # função que decifra o criptograma
    unpadder = padding.PKCS7(BLOCK_SIZE * 8).unpadder()
    res = True
    try:
        ptxt = unpadder.update(padded_msg) + unpadder.finalize()
    except ValueError:
        res = False
    return res # retorna apenas se houve ou não erro no "unpadding"
```

## *Padding oracle attack* (I)

A decifragem no modo CBC possibilita que o adversário, manipulando o penúltimo bloco do criptograma, controle o que é passado à funcionalidade de *unpadding*. Isso, combinado com um *padding oracle*, dá a possibilidade de o atacante atacar a cifra. (ver, e.g. https://www.nccgroup.com/research/cryptopals-exploiting-cbc-padding-oracles/)
